# Lecture Figure Guide

This guide explains how to read the figures produced by the lecture examples.

## Axis Types

| X-axis | Used in | Meaning |
| --- | --- | --- |
| degree `k` | `degree_distribution.png` | Empirical graph degree class. This checks graph loading and degree aggregation. |
| time | optimal-control, game, node-level, and hybrid plots | Simulated model time. These figures show system state evolution and computed controls/strategies. |
| vertical impulse markers | `hybrid_impulse.png` | Discrete intervention times in a hybrid continuous-discrete simulation. |

## Plotting Convention

- Continuous controls or strategies are time-indexed curves. They may vary smoothly with time, or appear constant in a smoke run if the chosen control level is constant.
- Impulse controls act only at discrete event times. They are drawn as vertical event lines, not as connected curves.
- Hybrid control combines continuous control with impulse interventions. In hybrid plots, state curves may jump at impulse times because a discrete map is applied between continuous ODE segments.
- State labels specify the aggregation level: node mean over all nodes, degree-weighted mean over degree classes, or a selected degree class `k`.

## Figure Types

| Figure pattern | What it shows | How to interpret it |
| --- | --- | --- |
| `*/degree_distribution.png` | Degree classes and `P(k)` | A smoke check that the input graph was parsed and converted into degree statistics. |
| `simple_*/degree_k_control.png` | Degree-weighted mean infection/control and selected degree-class states | A compact degree-k continuous optimal-control example. State curves identify whether they are degree-weighted means or a specific degree class. |
| `examples_*/degree_control.png` | Degree-level state and continuous-control trajectories | The system evolves under the computed continuous optimal control. |
| `examples_*/degree_game.png` | Degree-level infection plus continuous attack/defense strategies | The attacker and defender controls are continuous open-loop game strategies computed by the forward-backward sweep. |
| `examples_*/node_control.png` | Node-mean/max infection and node-mean continuous control | Node-level continuous optimal-control behavior on a reduced graph. |
| `examples_*/node_game.png` | Node-mean infection plus continuous attack/defense strategies | Node-level continuous differential-game behavior under computed strategies. |
| `examples_*/hybrid_impulse.png` | Node-level states, continuous control, and impulse magnitudes | A hybrid-control smoke test: the top panel shows node-level state summaries and impulse-time markers, while the bottom panel separates continuous control from discrete impulse magnitudes. |
| `simple_contact_sheet.png` and `examples_contact_sheet.png` | Visual indexes | A quick overview of the figures generated by a run. |

The lecture smoke runs are intentionally small. They are designed to verify that graph loading, degree aggregation, state equations, adjoint/control updates, plotting, and output paths work before moving to paper-level reference code.
