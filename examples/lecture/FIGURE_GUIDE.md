# Lecture Figure Guide

This guide explains how to read the figures produced by the lecture examples.

## Axis Types

| X-axis | Used in | Meaning |
| --- | --- | --- |
| degree `k` | `degree_distribution.png` | Empirical graph degree class. This checks graph loading and degree aggregation. |
| time | optimal-control, game, node-level, and hybrid plots | Simulated model time. These figures show system state evolution and computed controls/strategies. |
| vertical impulse markers | `hybrid_impulse.png` | Discrete intervention times in a hybrid continuous-discrete simulation. |

## Plotting Convention

- Curves represent continuous states, controls, or game strategies.
- Vertical lines represent discrete impulse interventions.
- In hybrid plots, state curves may jump at impulse times because a discrete map is applied between continuous ODE segments.

## Figure Types

| Figure pattern | What it shows | How to interpret it |
| --- | --- | --- |
| `*/degree_distribution.png` | Degree classes and `P(k)` | A smoke check that the input graph was parsed and converted into degree statistics. |
| `simple_*/degree_k_control.png` | Mean infection, mean control, and selected degree-class states | A compact degree-k optimal-control example. The state and control curves are plotted against time. |
| `examples_*/degree_control.png` | Degree-level state and control trajectories | The system evolves under the computed optimal control. |
| `examples_*/degree_game.png` | Degree-level infection, attack, and defense trajectories | The attacker and defender controls are open-loop game strategies computed by the forward-backward sweep. |
| `examples_*/node_control.png` | Node-level mean/max infection and mean control | Node-level optimal-control behavior on a reduced graph. |
| `examples_*/node_game.png` | Node-level infection, attack, and defense | Node-level differential-game behavior under computed strategies. |
| `examples_*/hybrid_impulse.png` | Mean/max infection, continuous control, and impulse magnitudes | A hybrid-control smoke test: the top panel shows state evolution and impulse-time markers, while the bottom panel separates continuous control from discrete impulse magnitudes. |
| `simple_contact_sheet.png` and `examples_contact_sheet.png` | Visual indexes | A quick overview of the figures generated by a run. |

The lecture smoke runs are intentionally small. They are designed to verify that graph loading, degree aggregation, state equations, adjoint/control updates, plotting, and output paths work before moving to paper-level reference code.
