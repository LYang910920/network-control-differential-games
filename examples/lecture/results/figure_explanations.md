# Lecture Example Figure Explanations

This file is generated by `run_all_lecture_examples.py` for the current run.

## How to read the x-axes

- `degree k`: the graph degree class. These plots check whether the input graph was loaded and aggregated correctly.
- `time`: the simulated model time. These plots show how the system state and the computed control or game strategies evolve.
- Continuous controls or strategies: time-indexed curves sampled on the simulation grid. Projected controls can have flat bound segments, and the generated examples use line styles and parameter choices that make their time variation visible.
- Impulse controls: vertical event lines at discrete intervention times. They are not connected continuous curves.
- Hybrid control: a combination of continuous control and impulse interventions. State trajectories may jump at impulse times.
- State labels specify the aggregation level: node mean over all nodes, degree-weighted mean over degree classes, or a selected degree class `k`.

| `*/fbs_convergence.png` | iteration | Maximum control/strategy change per forward-backward sweep step | Convergence diagnostic; the curve should decrease toward the stated tolerance. |
| `*/baseline_comparison.png` | scenario | Computed control/game strategies compared with simple baselines | Sanity check that the computed strategy is interpreted against no-control, constant-control, or unilateral zero-deviation game baselines. |

| `examples_contact_sheet.png` | mixed | Overview of companion examples | Visual index for degree, node, and hybrid smoke cases. |

The purpose of these runs is not full paper-scale reproduction. They are small, fast checks that the model-loading, state dynamics, adjoint/control update, plotting, and file-output paths all work.
