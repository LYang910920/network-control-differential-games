# Lecture Example Figure Explanations

This file is generated by `run_all_lecture_examples.py` for the current run.

## How to read the x-axes

- `degree k`: the graph degree class. These plots check whether the input graph was loaded and aggregated correctly.
- `time`: the simulated model time. These plots show how the system state and the computed control or game strategies evolve.
- Curves: continuous states, controls, or game strategies.
- Vertical lines: discrete impulse interventions. State trajectories may jump at these event times.

## Generated outputs

| Output pattern | X-axis | What it shows | Interpretation |
| --- | --- | --- | --- |
| `*/degree_distribution.png` | degree `k` | Empirical degree distribution `P(k)` | Smoke check for graph loading and degree aggregation. |
| `simple_*/degree_k_control.png` | time | Mean infection, mean control, and selected degree-class states | Degree-k optimal-control trajectory produced by the forward-backward sweep. |
| `examples_*/degree_control.png` | time | Degree-level infection and control | State evolution under the computed optimal control. |
| `examples_*/degree_game.png` | time | Degree-level infection, attack, and defense | Open-loop attacker-defender game trajectory under computed strategies. |
| `examples_*/node_control.png` | time | Node-level mean/max infection and mean control | Node-level optimal-control trajectory on the reduced graph. |
| `examples_*/node_game.png` | time | Node-level mean infection, attack, and defense | Node-level differential-game trajectory under computed strategies. |
| `examples_*/hybrid_impulse.png` | time | Mean/max infection, continuous control, and impulse magnitudes | Hybrid simulation with continuous dynamics in the top panel and discrete impulse interventions in the bottom panel. |
| `simple_contact_sheet.png` | mixed | Overview of simple degree-k runs | Visual index for the three simple smoke cases. |
| `examples_contact_sheet.png` | mixed | Overview of companion examples | Visual index for degree, node, and hybrid smoke cases. |

The purpose of these runs is not full paper-scale reproduction. They are small, fast checks that the model-loading, state dynamics, adjoint/control update, plotting, and file-output paths all work.
