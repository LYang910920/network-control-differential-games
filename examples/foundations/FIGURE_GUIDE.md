# Tutorial Figure Guide

This guide explains how to read the figures produced by the tutorial examples.

## Axis Types

| X-axis | Used in | Meaning |
| --- | --- | --- |
| degree `k` | `degree_distribution.png` | Empirical graph degree class. This checks graph loading and degree aggregation. |
| time | optimal-control, game, node-level, and hybrid plots | Simulated model time. These figures show system state evolution and computed controls/strategies. |
| network size | `degree_node_fbs_comparison_100_1000000.png` | Number of nodes in synthetic scale-free networks used for paired runtime timing. This figure uses log-scaled axes. |
| vertical impulse markers | `hybrid_impulse_trajectory.png` | Discrete intervention times in a hybrid continuous-discrete simulation. |

## Plotting Convention

- Continuous controls or strategies are time-indexed curves sampled on the simulation grid. Projected controls can have flat bound segments, and the generated examples use line styles and parameter choices that make their time variation visible.
- Impulse controls act only at discrete event times. They are drawn as vertical event lines, not as connected curves.
- Hybrid control combines continuous control with impulse interventions. In hybrid plots, state curves may jump at impulse times because a discrete map is applied between continuous ODE segments.
- State labels specify the aggregation level: node mean over all nodes, degree-weighted mean over degree classes, or a selected degree class `k`.

## Figure Types

| Figure pattern | What it shows | How to interpret it |
| --- | --- | --- |
| `*/degree_distribution.png` | Degree classes and `P(k)` | A smoke check that the input graph was parsed and converted into degree statistics. |
| `simple_*/degree_control_trajectory.png` | Degree-weighted mean infection/control and selected degree-class states | A compact degree-k continuous optimal-control example. State curves identify whether they are degree-weighted means or a specific degree class. |
| `companion_*/degree_control_trajectory.png` | Degree-level state and continuous-control trajectories | The system evolves under the computed continuous optimal control. |
| `companion_*/degree_game_trajectory.png` | Degree-level infection plus continuous attack/defense strategies | The attacker and defender controls are continuous open-loop game strategies computed by the forward-backward sweep. |
| `companion_*/node_control_trajectory.png` | Node-mean/max infection and node-mean continuous control | Node-level continuous optimal-control behavior on a reduced graph. |
| `companion_*/node_game_trajectory.png` | Node-mean infection plus continuous attack/defense strategies | Node-level continuous differential-game behavior under computed strategies. |
| `companion_*/hybrid_impulse_trajectory.png` | Node-level states, continuous control, and impulse magnitudes | A hybrid-control smoke test: the top panel shows node-level state summaries and impulse-time markers, while the bottom panel separates continuous control from discrete impulse magnitudes. |
| `*/fbs_convergence.png` | FBS residual curves | Maximum control or strategy change per forward-backward sweep iteration. These curves should decrease toward the numerical tolerance before interpreting the computed control. |
| `simple_*/degree_control_baseline_comparison.png` | Simple degree-k baseline comparison | Computed continuous control is compared with no-control, constant-control, and 75 random smooth-control baselines. |
| `companion_*/*_control_baseline_comparison.png` | One control model per baseline figure | Computed continuous control is compared with no-control, constant-control, and 75 random smooth-control baselines. |
| `companion_*/*_game_baseline_comparison.png` | One game model per baseline figure | Two unilateral panels: fixed computed attack with varied defense, and fixed computed defense with varied attack. Each panel compares the computed player strategy with zero and random strategies. |
| `scalability_degree_node_sf/degree_node_fbs_comparison_100_1000000.png` | Paired degree-level versus sparse node-level FBS runtime | Both curves use the same normalized SIS epidemic-control model, graph seeds, RK4 time grid, and FBS tolerance. The degree curve has one state/control/costate per observed degree class; the node curve has one state/control/costate per graph node and uses sparse adjacency products. The third panel checks that maximum degree grows with SF network size. |
| `*/parameter_summary.csv` | Parameter table for one smoke run | Concrete values for time horizon, grid size, propagation/recovery rates, control bounds, impulse settings, and random-baseline count. |
| `simple_contact_sheet.png` and `companion_contact_sheet.png` | Visual indexes | A quick overview of the figures generated by a run. The `companion_*` prefix is the historical output-folder name for the main foundation examples. |

The tutorial smoke runs are small by design. They verify that graph loading, degree aggregation, state equations, adjoint/control updates, plotting, and output paths work before moving to paper-level reference code.
