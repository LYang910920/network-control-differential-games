# Cost-Effective Hybrid Control Strategies for Dynamical Propaganda War Game

This repository contains the source code accompanying our paper:

> Xiaojuan Cheng, Lu-Xing Yang, Qingyi Zhu, Chenquan Gan, Xiaofan Yang, Gang Li.
> **"Cost-Effective Hybrid Control Strategies for Dynamical Propaganda War Game."**
> *IEEE Transactions on Information Forensics and Security*, vol. 19, pp. 9789–9802, 2024.
> DOI: [10.1109/TIFS.2024.3468903](https://doi.org/10.1109/TIFS.2024.3468903)
> IEEE Xplore: <https://ieeexplore.ieee.org/document/10695110>

## Overview

We model the conflict between two opposing propaganda campaigns (a **Red** team and a **Blue** team) on a heterogeneous social network as a **differential game** and design **cost-effective hybrid control strategies** that combine continuous and impulsive interventions. The game is solved as a degree-based mean-field system; the Nash-equilibrium strategy for each team is obtained by a forward–backward sweep over the state and adjoint equations.

The code implements:

- The degree-based propagation model with continuous controls `u_r(t)`, `u_b(t)` and impulse controls `v_r(τ_i)`, `v_b(τ_j)`.
- The forward state dynamics for both teams (`forwardRed`, `forwardBlue`).
- The adjoint (co-state) dynamics for both teams (`adjRed`, `adjBlue`, `backwardRed`, `backwardBlue`).
- The optimal strategy search at each iteration (`optimalStrategyRed`, `optimalStrategyBlue`) and the corresponding payoff functionals (`payoffRed`, `payoffBlue`).
- A random-strategy baseline used to empirically validate the Nash-equilibrium property of the computed strategy (`comparison.py`).

## Files

| File | Purpose |
| --- | --- |
| `propWar.py` | Main algorithm — forward / backward sweeps and the iterative search for the Nash-equilibrium hybrid strategies for both teams. Entry point of the experiment. |
| `demo_network.py` | Network utilities: load a graph from a `.csv` / `.edges` / `.mtx` edgelist (`graphs`, `graphs_tmx`) and compute its degree distribution (`my_degree`). |
| `comparison.py` | Empirical Nash-equilibrium check: with one team fixed to the Nash strategy obtained from `propWar.py`, randomly sample the other team's strategy `m` times and compare payoffs. |

## Datasets

The experiments use public Facebook and Twitter network datasets. **The dataset list and the corresponding download sources are described in the paper** — please refer to the paper for the exact networks used in each experiment and their original references. Datasets are not redistributed in this repository.

Place the downloaded network files under `./data/` (the paths at the top of each script's `__main__` block expect this layout):

```
./data/<network_files>
./result/
    result_new33/          # generated by propWar.py
    random/                # generated by comparison.py
    figs/                  # generated figures
```

Create `./result/result_new33/` and `./result/random/` before running, so that `df.to_csv(...)` has somewhere to write.

## Usage

**Step 1 — Compute the Nash-equilibrium hybrid strategy:**

```bash
python propWar.py
```

The two graph files at the top of `propWar.py`'s `__main__` (variables `fpr` and `fpb`) select the Red and Blue networks; the parameter block below them sets the model rates (β, γ, δ), unit costs (`c_r`, `c_b`), strategy bounds, pulse intervals, etc. Save lines for the per-iteration outputs (`pr_*.csv`, `cr_*.csv`, `ur_*.csv`, `vr_*.csv`, `jr_*.csv`, etc.) are present but commented out — uncomment them after creating the target directory if you want to persist results for the comparison step.

**Step 2 — Validate the Nash equilibrium against random strategies:**

```bash
python comparison.py
```

You will be prompted `which team you want to compare?` — enter `r` to perturb the Red team's strategy while keeping Blue at the Nash solution (loads `ub_*.csv` and `vb_*.csv` from `./result/result_new33/`), or `b` to do the opposite. The resulting payoff samples are saved to `./result/random/`.

The Python parameter block in `comparison.py` is currently set up for the **Twitter** dataset pair (`rt_barackobama.edges` + `rt-pol.txt`); swap the file paths and the corresponding parameter values if you want to run on Facebook networks instead.
